一、Python 基础
1. is 和 == 的区别？
== 比较两个对象的值是否相等。

is 比较两个对象是否是同一个对象（内存地址是否相同）。

示例：

python
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)   # True
print(a is b)   # False
2. Python 中的可变对象和不可变对象有哪些？
不可变对象：int, float, str, tuple, frozenset。修改后会创建新对象。

可变对象：list, dict, set，自定义类实例等。修改时内存地址不变。

3. 解释 Python 中的 *args 和 **kwargs
*args 用于接收任意数量的位置参数，以元组形式传入。

**kwargs 用于接收任意数量的关键字参数，以字典形式传入。

常用于函数定义中处理可变参数。

4. Python 的装饰器是什么？写一个简单的装饰器
装饰器是一个接受函数并返回新函数的可调用对象（通常是一个函数），用于在不修改原函数代码的情况下添加额外功能。

示例：

python
def timer(func):
    def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = func(*args, **kwargs)
        print(f"Time: {time.time()-start}")
        return result
    return wrapper

@timer
def my_func():
    time.sleep(1)
5. 什么是生成器？和迭代器有什么区别？
迭代器：实现了 __iter__() 和 __next__() 方法的对象，可以用 next() 逐个获取元素。

生成器：一种特殊的迭代器，使用 yield 关键字，函数调用时返回一个生成器对象，可以暂停和恢复执行。

区别：生成器写法更简洁，自动实现迭代器协议，且是懒惰求值的。

6. 解释 Python 中的 GIL（全局解释器锁）
GIL 是 CPython 解释器中的一个互斥锁，保证同一时刻只有一个线程执行 Python 字节码。

它简化了内存管理（引用计数），但限制了多线程的并行性。对于 CPU 密集型任务，多线程并不能充分利用多核，通常使用多进程或异步 IO 解决。

7. __new__ 和 __init__ 的区别
__new__ 是静态方法，用于创建并返回一个新的实例（控制实例创建过程），第一个参数是类本身。

__init__ 是实例方法，用于初始化已创建的对象，第一个参数是实例。

通常 __new__ 用在单例模式或不可变类的定制中。

二、数据结构与算法
8. 列表和元组的区别
列表可变，元组不可变。

列表通常用于同质数据，元组用于异质数据（但并非绝对）。

元组可作为字典的键，列表不行。

9. 字典的底层实现是什么？
Python 字典使用哈希表实现，通过哈希函数将键映射到存储桶，解决冲突通常用开放寻址法。

插入、查找、删除的平均时间复杂度为 O(1)。

10. 如何反转字符串/列表？
字符串反转：s[::-1]

列表反转：lst[::-1] 或 lst.reverse()（就地反转）

11. 请实现一个函数，判断一个字符串是否是回文串
python
def is_palindrome(s):
    s = s.lower()
    return s == s[::-1]
12. 请用 Python 实现一个单例模式（至少两种方式）
使用装饰器：

python
def singleton(cls):
    instances = {}
    def wrapper(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    return wrapper
使用 __new__：

python
class Singleton:
    _instance = None
    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super().__new__(cls)
        return cls._instance
使用元类：

python
class SingletonMeta(type):
    _instances = {}
    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]
三、面向对象
13. Python 支持多继承吗？如果多个父类有同名方法，如何确定调用顺序？
Python 支持多继承，方法解析顺序（MRO）遵循 C3 线性化算法，可通过 类名.__mro__ 查看。

调用时按照 MRO 顺序查找第一个匹配的方法。

14. 什么是 MRO？如何查看？
MRO 即方法解析顺序，用于确定在多继承中调用方法的查找顺序。

查看方式：类名.__mro__ 或 类名.mro()

15. @staticmethod 和 @classmethod 的区别
@staticmethod 定义静态方法，不接收隐式第一参数（self 或 cls），相当于定义在类中的普通函数。

@classmethod 定义类方法，接收第一个参数为类本身（cls），可以访问类属性和调用其他类方法。

16. 私有属性和方法是如何实现的？
Python 通过名称修饰（name mangling）实现私有：以双下划线开头的属性（如 __name）会被重命名为 _类名__name，从而达到“伪私有”的效果。

但实际仍可通过 _类名__name 访问，更多是约定而非强制。

四、高级特性
17. 什么是上下文管理器？如何使用 with 语句？
上下文管理器用于资源管理，如文件打开、锁获取释放，确保资源在使用后被正确清理。

需要实现 __enter__ 和 __exit__ 方法。

with open('file.txt') as f: 就是典型用法。

18. 什么是闭包？
闭包是指在一个内部函数中引用了外部函数作用域的变量，并且外部函数返回内部函数。

示例：

python
def outer(x):
    def inner(y):
        return x + y
    return inner
add5 = outer(5)
print(add5(3))   # 8
19. 什么是装饰器？你用过哪些内置装饰器？
装饰器已在前面提到。内置装饰器有：@property, @staticmethod, @classmethod, @functools.lru_cache 等。

20. 解释 yield 关键字的作用
yield 用于定义生成器函数，当函数执行到 yield 时，会返回一个值并暂停，下次调用 next() 时从暂停处继续执行。

21. 什么是 Python 的深浅拷贝？
浅拷贝（copy.copy）：创建一个新对象，但包含的子对象是原对象的引用。

深拷贝（copy.deepcopy）：递归地复制所有对象，新对象与原对象完全独立。

对于不可变对象，深浅拷贝通常无差别；对于可变对象，深拷贝更安全。

五、并发与异步
22. 进程、线程、协程的区别
进程：资源分配的最小单位，独立内存空间，切换开销大。

线程：CPU 调度的最小单位，共享进程内存，切换开销较小，但有 GIL 限制。

协程：用户态轻量级线程，由程序自己控制切换，开销极小，适合高并发 IO 场景。

23. 什么是 asyncio？简单示例
asyncio 是 Python 用于编写异步 IO 并发代码的库，使用 async/await 语法。

示例：

python
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

asyncio.run(say_hello())
六、内存管理与性能
24. Python 的内存管理机制
引用计数为主，标记-清除和分代回收为辅。

每个对象维护引用计数，为 0 时立即回收。

循环引用由垃圾回收器处理。

25. 如何优化 Python 程序的性能？
使用内置函数和数据结构（如 list 推导式、map、filter）。

避免循环中的属性访问，尽量局部变量。

使用生成器减少内存占用。

对 CPU 密集型任务使用多进程，IO 密集型使用异步或多线程。

使用 PyPy、Cython 等加速。

借助性能分析工具（cProfile）定位瓶颈。

七、常见陷阱
26. 默认参数的可变对象陷阱
默认参数在函数定义时只计算一次，如果默认参数是可变对象（如列表），多次调用会共享同一个对象。

应使用 None 作为默认值，在函数内部创建新对象。

27. 列表推导式中的变量泄露（Python 2 中，Python 3 已修复）
Python 2 中列表推导式的循环变量会泄露到外部作用域，Python 3 中不会。

28. 类属性与实例属性同名时的访问顺序
实例属性优先于类属性。如果实例没有该属性，才去类中查找。

八、开放性问题
29. 谈谈你对 Python 的 GIL 的理解，以及如何规避它的影响？
GIL 使得多线程无法并行执行 CPU 密集型任务，但对 IO 密集型影响不大。

规避方法：使用多进程（multiprocessing）、使用 C 扩展（如 numpy 释放 GIL）、使用异步编程。

30. 解释一下 Python 的元类（metaclass）以及应用场景
元类是创建类的类，默认是 type。可以通过自定义元类来拦截类的创建过程，动态修改类定义。

应用：ORM 框架（如 Django 模型）、自动注册子类、单例模式、属性验证等。
